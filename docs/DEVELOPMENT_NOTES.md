# Development Notes

These notes keep lyramusic changes easier to review.

## Local workflow

- Install dependencies from the lockfile when one is present.
- Run the existing build or lint command before opening a pull request.
- Keep UI changes and data changes in separate pull requests when possible.

## Review notes

- Include screenshots for visual changes.
- Mention any manual verification that was performed.
- Call out follow-up work instead of hiding it in code comments.
